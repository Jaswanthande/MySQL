CONSTRAINTS

CONSTRAINTS ARE RULES APPLIED TO TABLE COLUMNS IN MYSQL TO ENSURE THAT ONLY VALID, ACCURATE, AND CONSISTENT DATA IS STORED IN THE DATABASE.

IN SIMPLE WORDS

CONSTRAINTS ARE RULES THAT CONTROL THE DATA ENTERED INTO A TABLE.

WHY DO WE USE CONSTRAINTS?

TO MAINTAIN DATA INTEGRITY.

TO PREVENT INVALID DATA ENTRY.

TO AVOID DUPLICATE RECORDS.

TO ENSURE RELATIONSHIPS BETWEEN TABLES.

TO MAKE THE DATABASE RELIABLE AND CONSISTENT.

TYPES OF CONSTRAINTS

NOT NULL – DOES NOT ALLOW NULL VALUES.

UNIQUE – DOES NOT ALLOW DUPLICATE VALUES.

PRIMARY KEY – UNIQUELY IDENTIFIES EACH ROW IN A TABLE. IT CANNOT CONTAIN NULL OR DUPLICATE VALUES.

COMPOSITE KEY – A COMBINATION OF TWO OR MORE COLUMNS USED TO UNIQUELY IDENTIFY EACH ROW.

FOREIGN KEY – ESTABLISHES A RELATIONSHIP BETWEEN TWO TABLES AND MAINTAINS REFERENTIAL INTEGRITY.

A FOREIGN KEY CREATES A RELATIONSHIP BETWEEN TWO TABLES.
IT REFERENCES THE PRIMARY KEY OF ANOTHER TABLE.
IT PREVENTS INSERTING INVALID VALUES THAT DO NOT EXIST IN THE PARENT TABLE.
IT MAINTAINS DATA CONSISTENCY AND REFERENTIAL INTEGRITY.
ONE PRIMARY KEY CAN BE REFERENCED BY MANY FOREIGN KEYS


WHAT ARE REFERENTIAL ACTIONS

REFERENTIAL ACTIONS ARE RULES THAT SPECIFY HOW A DATABASE HANDLES UPDATES OR DELETES IN A PARENT TABLE WHEN RELATED RECORDS EXIST IN A CHILD TABLE.
