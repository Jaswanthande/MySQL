# FRONTEND

FRONTEND IS THE VISUAL AND INTERACTIVE LAYER OF A WEBSITE OR APPLICATION THAT USERS DIRECTLY SEE AND INTERACT WITH THROUGH A WEB BROWSER OR MOBILE APPLICATION.

IT IS ALSO CALLED THE CLIENT-SIDE BECAUSE IT RUNS ON THE USER'S DEVICE.

## RESPONSIBILITIES OF THE FRONTEND

1. DISPLAYS WEB PAGES AND THE USER INTERFACE (UI).
2. ACCEPTS USER INPUT, SUCH AS FORMS, BUTTONS, AND SEARCH BOXES.
3. VALIDATES USER INPUT BEFORE SENDING IT TO THE SERVER.
4. CALLS BACKEND APIS TO FETCH OR SEND DATA.
5. DISPLAYS THE RESPONSES RECEIVED FROM THE BACKEND.
6. PROVIDES ANIMATIONS AND INTERACTIVE EFFECTS.

TO BUILD THE FRONTEND LAYER, WE USE TECHNOLOGIES SUCH AS HTML, CSS, JAVASCRIPT, AND BOOTSTRAP.

---

# BACKEND

BACKEND IS THE SERVER-SIDE PART OF AN APPLICATION.

USERS CANNOT SEE IT DIRECTLY, BUT IT PERFORMS ALL THE BUSINESS LOGIC, SECURITY, DATA PROCESSING, AND DATABASE OPERATIONS. IT ACTS AS THE BRAIN OF THE APPLICATION.

## RESPONSIBILITIES OF THE BACKEND

1. PROCESSES CLIENT REQUESTS.
2. IMPLEMENTS BUSINESS LOGIC.
3. CONNECTS TO THE DATABASE.
4. PERFORMS CRUD (CREATE, READ, UPDATE, DELETE) OPERATIONS.
5. AUTHENTICATES AND AUTHORIZES USERS.
6. GENERATES REPORTS.
7. RETURNS RESPONSES TO THE FRONTEND.

SOME POPULAR PROGRAMMING LANGUAGES USED TO BUILD THE BACKEND ARE PYTHON, JAVA, C#, C++, JAVASCRIPT (NODE.JS), PHP, AND GO.

---

# API (APPLICATION PROGRAMMING INTERFACE)

AN API (APPLICATION PROGRAMMING INTERFACE) IS A SET OF RULES AND PROTOCOLS THAT ALLOWS TWO DIFFERENT APPLICATIONS OR SYSTEMS TO COMMUNICATE AND EXCHANGE DATA WITHOUT EXPOSING THEIR INTERNAL IMPLEMENTATION.

IT ACTS AS A BRIDGE BETWEEN THE FRONTEND AND THE BACKEND, OR BETWEEN TWO DIFFERENT SOFTWARE SYSTEMS.

## ADVANTAGES OF APIS

1. ENABLES COMMUNICATION BETWEEN DIFFERENT SYSTEMS.
2. PROMOTES CODE REUSABILITY.
3. IS PLATFORM-INDEPENDENT.
4. SPEEDS UP APPLICATION DEVELOPMENT.
5. SUPPORTS INTEGRATION WITH THIRD-PARTY SERVICES.
6. IMPROVES SCALABILITY.
7. MAKES APPLICATIONS EASIER TO MAINTAIN.

# STORAGE AREAS

AS PART OF AN APPLICATION, WE NEED TO STORE DATA SUCH AS CUSTOMER INFORMATION, BILLING INFORMATION, CALL RECORDS, AND OTHER DETAILS.

TO STORE THIS DATA, WE REQUIRE STORAGE AREAS.

THERE ARE TWO TYPES OF STORAGE AREAS:

## 1. TEMPORARY STORAGE AREA

TEMPORARY STORAGE AREAS ARE MEMORY LOCATIONS WHERE DATA IS STORED TEMPORARILY.

EXAMPLE: ALL JVM MEMORY AREAS, SUCH AS THE HEAP, STACK, METHOD AREA, PC REGISTER, AND NATIVE METHOD STACK. ONCE THE JVM IS SHUT DOWN, ALL THESE MEMORY AREAS ARE CLEARED AUTOMATICALLY.

## 2. PERMANENT STORAGE AREA

A PERMANENT STORAGE AREA IS ALSO KNOWN AS A PERSISTENT STORAGE AREA, WHERE DATA IS STORED PERMANENTLY.

EXAMPLES: FILE SYSTEMS, DATABASES, DATA WAREHOUSES, ETC.

---

# FILE MANAGEMENT SYSTEM (FMS)

A FILE MANAGEMENT SYSTEM (FMS) IS A SYSTEM IN WHICH DATA IS STORED AS FILES IN THE OPERATING SYSTEM.

EACH APPLICATION MUST MANAGE ITS OWN DATA STORAGE, RETRIEVAL, AND UPDATES. THESE OPERATIONS ARE PERFORMED USING THE SERVICES PROVIDED BY THE OPERATING SYSTEM.

FILE MANAGEMENT SYSTEMS ARE BEST SUITED FOR STORING A SMALL AMOUNT OF DATA.

EXAMPLE: A LIBRARY MANAGEMENT SYSTEM MAY STORE DATA IN SEPARATE TEXT FILES SUCH AS BOOKS.TXT, MEMBERS.TXT, ETC.

## LIMITATIONS OF FILE MANAGEMENT SYSTEM

1. DATA REDUNDANCY          – THE SAME DATA IS STORED IN MULTIPLE FILES.
2. DATA INCONSISTENCY       – UPDATES MADE IN ONE FILE MAY NOT BE REFLECTED IN OTHER FILES.
3. POOR DATA SECURITY       – THERE IS NO PROPER ACCESS CONTROL.
4. DIFFICULT DATA RETRIEVAL – SEARCHING FOR DATA REQUIRES CUSTOM PROGRAMS.
5. DATA INTEGRITY ISSUES    – THERE ARE NO CONSTRAINTS SUCH AS UNIQUE ROLL NUMBERS.
6. SCALABILITY ISSUES       – IT BECOMES DIFFICULT TO MANAGE DATA AS IT GROWS.

TO OVERCOME THESE LIMITATIONS, WE USE A DATABASE.

---

# DATABASE MANAGEMENT SYSTEM (DBMS)

A DATABASE MANAGEMENT SYSTEM (DBMS) IS A COLLECTION OF SOFTWARE PROGRAMS THAT ENABLE USERS TO CREATE, MANAGE, AND MANIPULATE DATABASES.

IT ACTS AS AN INTERFACE BETWEEN THE USER AND THE DATABASE.

## ADVANTAGES OF DBMS

1. CAN STORE A LARGE AMOUNT OF DATA.
2. PROVIDES QUERY LANGUAGE SUPPORT, MAKING DATABASE OPERATIONS EASY TO PERFORM.
3. REQUIRES USER AUTHENTICATION, SUCH AS A USERNAME AND PASSWORD, TO ACCESS THE DATABASE, THEREBY IMPROVING SECURITY.
4. STORES DATA IN THE FORM OF TABLES.
5. SUPPORTS NORMALIZATION TECHNIQUES TO REDUCE DATA REDUNDANCY.
6. SUPPORTS CONSTRAINTS SUCH AS PRIMARY KEY, FOREIGN KEY, UNIQUE KEY, AND CHECK CONSTRAINT TO MAINTAIN DATA INTEGRITY.
7. REDUCES DATA DUPLICATION AND DATA INCONSISTENCY.

## LIMITATIONS OF DBMS

1. A TRADITIONAL DATABASE CANNOT STORE OR PROCESS MASSIVE VOLUMES OF DATA EFFICIENTLY.
2. IT MAINLY SUPPORTS STRUCTURED (TABULAR OR RELATIONAL) DATA.
3. IT DOES NOT PROVIDE EFFICIENT SUPPORT FOR UNSTRUCTURED OR SEMI-STRUCTURED DATA SUCH AS VIDEOS, AUDIO FILES, IMAGES, XML FILES, OR JSON DOCUMENTS.

TO OVERCOME THESE LIMITATIONS, WE USE MORE ADVANCED STORAGE TECHNOLOGIES SUCH AS DATA WAREHOUSES, BIG DATA PLATFORMS, AND NOSQL DATABASES.
