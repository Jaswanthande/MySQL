SQL IDENTIFIES

SQL IDENTIFIES ARE NAMES USED TO IDENTIFY DATABASE OBJECT SUCH AS
DATABASE NAMES,TABLES NAMES,COLUMN NAMES,VIEW NAMES,IDEX NAMES,CONSTRAINTS,TRIGGERS,STORED  PROCEDURES.. ETC

WHILE WE ARE DECLARING IDEBTIFIERS WE HAVE TO FOLLOWS SOME RULES AND REGULATIONS
THE ONLY ALLOWED CHARACTER TO DEFINE IDENTIFIERS a-z, A-Z 0R 0 TO 9,_,&
MUST BE BEGIN WITH LETTER OR UNDERSCORE(_)
WE CAN'T DELCARE IDENTIFIERS WITH RESERVED WORDS
SPACES ARE NOT CONSIDERED TO DECLARE IDENTIFIERS
IDENTIFIERS ARE CASE SENSITIVE
MAX LENGTH TO DECLARE IDENTIFIERS ARE 15 CHARACTERS

SQL STRUCTURED QUERY LANGUAGE

SQL (STRUCTURED QUERY LANGUAGE) IS STANDARDIZED PROGRAMMING LANGUAGE SPECIALLY DESGINED FOR MANAGING AND MANIPULATING RELATIONAL DATABASE. IT PROVIDES A DECLARATIVE SYNTAX THAT ALLOWS USERS
TO DEFINE WHAT DAT THEY WANT TO RETRIVE,INSERT,UPDATE, OR DELETE WITHOUT SPECIFYING EXACTLY HOW DATABASE SHOULD EXECUTE THESE OPERATIONS


SQL COMMANDS

Command					 Type				Purpose	Commands
DDL						Define database objects			CREATE, ALTER, DROP, TRUNCATE, RENAME
DML						Manipulate data			   	    INSERT, UPDATE, DELETE, MERGE
DQL						Retrieve data				   	     SELECT
DCL						Control permissions			  	  GRANT, REVOKE
TCL						Manage transactions			     COMMIT, ROLLBACK, SAVEPOINT

DDL (DATA DEFINITION LANGUAGE)
DDL (DATA DEFINITION LANGUAGE) IS USED TO DEFINE, CREATE, MODIFY, AND DELETE THE STRUCTURE OF DATABASE OBJECTS SUCH AS TABLES, DATABASES, INDEXES, AND SCHEMAS.

CREATE:		 BY USING THESE COMMAND TO CREATE A NEW DATABASES,TABLES,VIEWS,INDEX.. ETC
ALTER:  	 BY USING ALTER COMMAND TO WE CAN MODIFY THE STRUCTURE OF AN EXISTING TABLES. LIKE(ADD COLUMN,DROP COLUMN,RENAME COLUMN,MODIFY DATATYPE,ADD CONSTARINTS)
DROP:		 DELETE AN ENTIRE DATABASE OBJECT PERMINENTLY LIKE (DATABASE,TABLES,VIEW,INDEX.. ETC)
TRUNCATE:	 BY USING TRUNCATE COMMAND DELETE ALL RECORDS FROM THE TABLE WITHOUT DELETING THE TABLE
RENAME:		 IT IS USED TO RENAME THE DATABASE OBJECT NAMES

DML (DATA MANIPULATION LANGUAGE)
DML (DATA MANIPULATION LANGUAGE) IS USED TO INSERT, UPDATE, DELETE, AND MODIFY THE DATA STORED IN DATABASE TABLES.

INSERT:		BY USING INSERT COMMAND WE CAN INSERT SOME RECORDS INTO TO A TABLE
UPDATE:		BY USING THE UPDATE COMMAND WE CAN UPDATE THE RECORDS BASED ON SOME CONDITIONS
DELETE:		USED TO REMOVE ONE OR MORE RECORDS (ROWS) FROM A TABLE BASED ON A SPECIFIED CONDITION.
LOCK:		CONTORLS CONCURRENCY BY LOCKING A TABLE  RESTRICT ACCESSES FROM THE OTHER TRANSACTION WHILE IT IS BEGIN MODIFIED

DCL (DATA CONTROL LANGUAGE)
DCL (DATA CONTROL LANGUAGE) IS USED TO CONTROL USER ACCESS AND PERMISSIONS IN A DATABASE BY GRANTING OR REVOKING.

GRANT:		GRANT IS A DCL COMMAND USED TO GIVE USERS OR ROLES PERMISSION TO ACCESS OR PERFORM OPERATIONS ON DATABASE OBJECTS.
REVOKE:		REVOKE IS A DCL COMMAND USED TO REMOVE PREVIOUSLY GRANTED PERMISSIONS OR PRIVILEGES FROM USERS OR ROLES.

TCL (TRANSACTION CONTROL LANGUAGE)
IS USED TO MANAGE DATABASE TRANSACTIONS AND ENSURE DATA CONSISTENCY BY SAVING OR UNDOING CHANGES.

COMMIT: 	COMMIT IS USED TO PERMANENTLY SAVE ALL CHANGES MADE DURING THE CURRENT TRANSACTION TO DATABASE MAKING THEM VISIABLE TI OTHER USERS.
ROLLBACK:	ROLLBACK IS USED TO UNDO CHANGES MADE DURING THE CURRENT TRANSACTION BEFORE THEY ARE COMMITTED.
SAVEPOINT:	SAVEPOINT IS USED TO CREATE A POINT WITHIN A TRANSACTION TO WHICH YOU CAN ROLLBACK LATER.

DQL (DATA QUERY LANGUAGE) 
DQL (DATA QUERY LANGUAGE) IS USED TO RETRIEVE, VIEW, AND FETCH DATA FROM DATABASE TABLES.

SELECT:		 SELECT IS USED TO RETRIEVE DATA FROM ONE OR MORE TABLES BASED ON SPECIFIED CONDITIONS.

HOW TO CREATE A DATABASE
SYNTAX:		CREATE DATABASE DATABASE_NAME;
EXAMPLE		CREATE DATABASE JASWANTH;

HOW TO DROP A DATABASE
SYNTAX		DROP DATABASE DATABASE_NAME;
EXAMPLE		DROP DATABASE JASWANTH;

HOW TO CREATE A TABLE
CREATE TABLE TABLE_NAME(
COLUMN_NAME DATATYPE(SIZE) CONSTRIANT,
COLUMN_NAME DATATYPE(SIZE),
COLUMN_NAME DATATYPE(SIZE),
COLUMN_NAME DATATYPE(SIZE),
);

EXAMPLE:
 CREATE TABLE EMPLOYESS(
    -> EMP_ID CHAR(5),
    -> FNAME VARCHAR(30),
    -> LNAME VARCHAR(100),
    -> AGE INT,
    -> DOJ DATE,
    -> ADDRESS TINYTEXT,
    -> DEPT VARCHAR(20)
    -> );

HOW TO SEE THAT WAS CREATED OR NOT
SYNTAX		SHOW TABLES;

HOW TO SEE COLUMNS OF THE TABLES
SYNTAX		SHOW COLUMNS FROM TABLE_NAME;
EXAMPLE		SHOW COLUMNS FROM EMPLOYESS;
