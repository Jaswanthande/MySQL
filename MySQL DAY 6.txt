WHEN A ROW IS DELETED FROM A TABLE, THE AUTO_INCREMENT PRIMARY KEY DOES NOT REUSE THE DELETED ID.

IF THE DELETED ROW HAD PRODUCT_ID = 3, THE NEXT INSERTED ROW WILL NOT GET PRODUCT_ID = 3. INSTEAD, THE AUTO_INCREMENT VALUE CONTINUES WITH THE NEXT AVAILABLE NUMBER.

EXAMPLE:
EXISTING IDS: 1, 2, 3
DELETE PRODUCT_ID = 3
INSERT A NEW ROW
NEW PRODUCT_ID = 4 (NOT 3)

GRANT
GRANT PRIVILAGE ON TABLE_NAME TO "USERNAME"

REVOKE
REVOKE PRIVILAGE ON TABLE_NAME FROM "USERNAME"
