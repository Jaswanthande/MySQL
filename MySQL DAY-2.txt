DIFFERENCES BETWEEN FILE MANAGEMENT SYSTEM (FMS) AND DATABASE MANAGEMENT SYSTEM (DBMS)

FILE MANAGEMENT SYSTEM (FMS)			 DATABASE MANAGEMENT SYSTEM (DBMS)

1.HIGH DATA REDUNDANCY (DUPLICATE DATA)				REDUCES DATA REDUNDANCY

2.DIFFICULT TO MAINTAIN AND UPDATE				EASY TO MAINTAIN AND UPDATE

3.LIMITED SECURITY						PROVIDES STRONG SECURITY

4.DATA SHARING IS DIFFICULT					MULTIPLE USERS CAN ACCESS DATA SIMULTANEOUSLY

5.NO RELATIONSHIP BETWEEN FILES					SUPPORTS RELATIONSHIPS BETWEEN TABLES

6.BACKUP AND RECOVERY ARE DIFFICULT				PROVIDES BACKUP AND RECOVERY FEATURES

7.SUITABLE FOR SMALL APPLICATIONS				SUITABLE FOR SMALL, MEDIUM, AND LARGE APPLICATIONS


WHAT IS THE DIFFERENCE BETWEEN A FILE SYSTEM AND A DBMS?

FILE SYSTEM:
A FILE SYSTEM STORES DATA IN SEPARATE FILES. IT HAS HIGH DATA REDUNDANCY, LIMITED SECURITY, DIFFICULT DATA SHARING, AND DOES NOT MAINTAIN RELATIONSHIPS BETWEEN DATA.

DBMS:
A DATABASE MANAGEMENT SYSTEM (DBMS) STORES DATA IN A STRUCTURED MANNER USING TABLES. IT REDUCES DATA REDUNDANCY, PROVIDES STRONG SECURITY, SUPPORTS MULTIPLE USERS, MAINTAINS RELATIONSHIPS BETWEEN TABLES, AND ENSURES DATA CONSISTENCY AND INTEGRITY.

WHAT IS DATA?
DATA IS A COLLECTION OF RAW FACTS OR FIGURES WITHOUT ANY CONTEXT OR MEANING.
Example: 111, JASSU, VIZAG

WHAT IS A FIELD?
A FIELD IS THE SMALLEST UNIT OF DATA IN A DATABASE. IT IS ALSO CALLED A COLUMN OR ATTRIBUTE.
Example: EMP_ID, EMP_ADDRESS

WHAT IS A RECORD?
A RECORD IS A COLLECTION OF RELATED FIELDS. IT IS ALSO CALLED A ROW OR TUPLE.
Example: 101, JASSU, GAJUWAKA

WHAT IS A DATABASE?
A DATABASE IS AN ORGANIZED COLLECTION OF RELATED DATA OR RECORDS STORED TOGETHER FOR EASY ACCESS, MANAGEMENT, AND UPDATION.
Example: AN EMPLOYEE DATABASE CONTAINS ALL EMPLOYEE DETAILS.

WHAT IS DBMS?
A DATABASE MANAGEMENT SYSTEM (DBMS) IS SOFTWARE USED TO CREATE, STORE, RETRIEVE, UPDATE, DELETE, AND MANAGE DATABASES.
CLIENT-SERVER ARCHITECTURE

CLIENT-SERVER ARCHITECTURE CONSISTS OF THREE MAIN COMPONENTS:
CLIENT
SERVER
PROTOCOL

CLIENT
THE CLIENT SENDS REQUESTS TO THE SERVER AND RECEIVES RESPONSES FROM THE SERVER.

SERVER
THE SERVER IS SOFTWARE THAT RECEIVES REQUESTS FROM CLIENTS.
IF THE REQUEST IS VALID, THE SERVER PROCESSES IT AND RETURNS THE REQUIRED RESPONSE.
IF THE REQUEST IS INVALID, THE SERVER RETURNS AN ERROR MESSAGE.

PROTOCOL
A PROTOCOL IS A SET OF RULES AND REGULATIONS THAT DEFINES HOW DATA IS COMMUNICATED BETWEEN THE CLIENT AND THE SERVER.
ITS ROLE IS TO TRANSFER REQUESTS FROM THE CLIENT TO THE SERVER AND RESPONSES FROM THE SERVER TO THE CLIENT.

DATA MODELS IN DBMS
A DATA MODEL DEFINES HOW DATA IS STORED, ORGANIZED, AND MANIPULATED IN A DATABASE.

PHYSICAL DATA MODEL
DESCRIBES HOW DATA IS PHYSICALLY STORED IN STORAGE DEVICES.
Audience: DATA ADMINISTRATORS

LOGICAL DATA MODEL
DEFINES THE STRUCTURE OF DATA (ENTITIES, ATTRIBUTES, AND RELATIONSHIPS) AND IS INDEPENDENT OF PHYSICAL STORAGE.
Audience: DEVELOPERS AND DATABASE ARCHITECTS

HIERARCHICAL DATA MODEL
DATA IS ORGANIZED IN A TREE STRUCTURE WITH A PARENT-CHILD RELATIONSHIP.
Example: COMPANY → DEPARTMENTS → EMPLOYEES

NETWORK DATA MODEL
DATA IS ORGANIZED AS A GRAPH WITH RECORDS (NODES) AND RELATIONSHIPS (EDGES). A RECORD CAN HAVE MULTIPLE PARENTS.
Example: A STUDENT CAN ENROLL IN MULTIPLE COURSES, AND A COURSE CAN BE TAUGHT BY MULTIPLE PROFESSORS.

RELATIONAL DATA MODEL (RDBMS)
DATA IS STORED IN TABLES (RELATIONS) CONSISTING OF ROWS (TUPLES) AND COLUMNS (ATTRIBUTES). RELATIONSHIPS BETWEEN TABLES ARE ESTABLISHED USING KEYS.

Example:
STUDENT (ID, NAME, MARKS, DEPT_ID)
DEPARTMENT (DEPT_ID, DEPT_NAME)
